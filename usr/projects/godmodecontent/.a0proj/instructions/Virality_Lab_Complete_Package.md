## Feature Set

### Core Features (MVP)

#### 1. Viral Video Analyzer
**Input:** Video URL (YouTube, TikTok, Instagram, Twitter/X)
**Output:** Complete viral breakdown

**Analysis Dimensions:**
- **Quantitative Metrics**
  - View count & velocity (views/hour in first 24h, 48h, 7d)
  - Engagement rate (likes, comments, shares vs. views)
  - Engagement velocity (how fast engagement accumulated)
  - Share-to-view ratio (viral coefficient)
  - Comment sentiment analysis
  - Platform-specific metrics (TikTok completion rate, YouTube CTR, etc.)

- **Content Analysis (AI-Powered)**
  - Hook breakdown (first 3 seconds frame-by-frame)
  - Narrative structure (setup, conflict, resolution, twist)
  - Emotional arc mapping
  - Pattern identification (curiosity gap, controversy, transformation, etc.)
  - Audio/music analysis (trending sounds, original audio)
  - Visual style (editing pace, transitions, text overlays)
  - Caption/title formula extraction
  - Hashtag strategy analysis

- **Contextual Factors**
  - Posting time analysis
  - Trend timing (early/peak/late to trend)
  - Creator authority/history
  - Cross-platform presence
  - Algorithm timing factors

#### 2. Virality Score (1-100)
Weighted scoring system across dimensions:

| Factor | Weight | Description |
|--------|--------|-------------|
| Hook Strength | 20% | First 3-second retention prediction |
| Emotional Trigger | 15% | Intensity of emotional response |
| Shareability | 15% | "I need to send this to someone" factor |
| Trend Alignment | 15% | Relevance to current cultural moment |
| Engagement Velocity | 10% | Speed of engagement accumulation |
| Completion Rate | 10% | Predicted watch-through rate |
| Comment Potential | 10% | Likelihood of comment engagement |
| Platform Optimization | 5% | Technical optimization for platform |

**Score Interpretation:**
- 90-100: Viral Guaranteed (0.1% of videos)
- 75-89: High Viral Potential (1% of videos)
- 60-74: Moderate Viral Potential (5% of videos)
- 40-59: Average Performance Expected (30% of videos)
- Below 40: Low Viral Potential (64% of videos)

#### 3. Reverse-Engineering Engine
Maps viral elements to replicable templates:

**Output Structure:**
```
VIRAL ELEMENT MAPPING

Hook Type: Curiosity Gap
├── Original: "I found something in my walls that shouldn't be there..."
├── Pattern: [Unexpected discovery] + [Ominous implication] + [Cliffhanger]
├── Your Version: "[Unexpected discovery in your niche] + [Why it matters] + [Cliffhanger]"
└── Examples for your niche: [3 customized examples]

Emotional Trigger: Fear + Curiosity
├── Primary: Fear of the unknown
├── Secondary: Need to know resolution
└── How to apply: [Specific instructions]

Narrative Structure: Mystery Box
├── Setup (0-3s): Present the mystery
├── Build (3-15s): Add details, increase tension
├── Payoff (15-30s): Reveal (or withhold for part 2)
└── Template: [Fill-in-the-blank structure]
```

#### 4. Replication Protocol Generator
Step-by-step guide customized to user's niche:

**Protocol Sections:**
1. **Pre-Production Checklist**
   - Equipment needed
   - Location/setting requirements

   - Script template

2. **Hook Formula**
   - Exact opening line template

   - Editing style to match

   - Sound design notes

5. **Publishing Strategy**
   - Optimal posting time

   - Hashtag strategy
   - Cross-posting plan

Professional downloadable report including:

## Database Schema

```sql
-- Users
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT UNIQUE NOT NULL,
  name TEXT,
  avatar_url TEXT,
  niche TEXT,
  platforms TEXT[], -- ['youtube', 'tiktok', 'instagram']
  subscription_tier TEXT DEFAULT 'free',
  subscription_status TEXT DEFAULT 'active',
  stripe_customer_id TEXT,
  analyses_used_this_month INT DEFAULT 0,
  analyses_limit INT DEFAULT 5,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Analyses
CREATE TABLE analyses (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  video_url TEXT NOT NULL,
  platform TEXT NOT NULL, -- 'youtube', 'tiktok', 'instagram', 'twitter'
  video_id TEXT NOT NULL, -- Platform-specific video ID
  
  -- Video Metadata
  title TEXT,
  description TEXT,
  thumbnail_url TEXT,
  creator_name TEXT,
  creator_followers INT,
  duration_seconds INT,
  published_at TIMESTAMPTZ,
  
  -- Metrics at time of analysis
  views INT,
  likes INT,
  comments INT,
  shares INT,
  engagement_rate FLOAT,
  
  -- AI Analysis Results
  virality_score INT, -- 1-100
  score_breakdown JSONB, -- {hook: 85, emotion: 72, ...}
  
  hook_analysis JSONB,
  -- {
  --   type: "curiosity_gap",
  --   strength: 92,
  --   text: "...",
  --   pattern: "...",
  --   improvements: [...]
  -- }
  
  emotional_triggers TEXT[],
  narrative_structure TEXT,
  viral_elements JSONB[], -- Array of identified elements
  
  transcript TEXT,
  key_moments JSONB[], -- [{timestamp, description, importance}]
  
  -- Replication Protocol
  replication_protocol JSONB,
  -- {
  --   hook_template: "...",
  --   structure: [...],
  --   checklist: [...],
  --   publishing: {...}
  -- }
  
  -- Export
  pdf_url TEXT,
  
  -- Metadata
  analysis_version TEXT DEFAULT '1.0',
  processing_time_ms INT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Hook Library
CREATE TABLE hooks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  analysis_id UUID REFERENCES analyses(id),
  text TEXT NOT NULL,
  type TEXT, -- 'curiosity_gap', 'controversy', 'transformation', etc.
  platform TEXT,
  niche TEXT,
  virality_score INT,
  engagement_rate FLOAT,
  embedding VECTOR(1536), -- For similarity search
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Templates (Replication protocols that work)
CREATE TABLE templates (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  description TEXT,
  category TEXT, -- 'hook', 'structure', 'format'
  niche TEXT,
  platform TEXT,
  template_content JSONB,
  usage_count INT DEFAULT 0,
  success_rate FLOAT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- User Saved Items
CREATE TABLE saved_analyses (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  analysis_id UUID REFERENCES analyses(id) ON DELETE CASCADE,
  notes TEXT,
  tags TEXT[],
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, analysis_id)
);

-- Usage Tracking
CREATE TABLE usage_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  action TEXT, -- 'analysis', 'pdf_export', 'protocol_generation'
  metadata JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Indexes
CREATE INDEX idx_analyses_user ON analyses(user_id, created_at DESC);
CREATE INDEX idx_analyses_platform ON analyses(platform);
CREATE INDEX idx_analyses_score ON analyses(virality_score DESC);
CREATE INDEX idx_hooks_type ON hooks(type);
CREATE INDEX idx_hooks_niche ON hooks(niche);
CREATE INDEX idx_hooks_embedding ON hooks USING ivfflat (embedding vector_cosine_ops);

-- RLS Policies
ALTER TABLE users ENABLE ROW LEVEL SECURITY;
ALTER TABLE analyses ENABLE ROW LEVEL SECURITY;
ALTER TABLE saved_analyses ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users read own data" ON users FOR SELECT USING (auth.uid() = id);
CREATE POLICY "Users update own data" ON users FOR UPDATE USING (auth.uid() = id);
CREATE POLICY "Users manage own analyses" ON analyses FOR ALL USING (auth.uid() = user_id);
CREATE POLICY "Users manage saved" ON saved_analyses FOR ALL USING (auth.uid() = user_id);
CREATE POLICY "Hooks readable by all" ON hooks FOR SELECT TO authenticated USING (true);
CREATE POLICY "Templates readable by all" ON templates FOR SELECT TO authenticated USING (true);
```
